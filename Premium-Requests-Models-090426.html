01 Topic 1

Premium Requests

Every interaction with advanced models or agent features consumes premium requests from your plan's monthly quota.

  • What counts as premium — chat, inline completions with premium models, and agent-mode actions each consume requests.
  • Monthly quotas — Individual plan includes 300/month; Business & Enterprise get larger pools per seat.
  • Unlimited base model — GPT-4o mini completions and chat remain unlimited and don't consume premium quota.
  • Overage options — organisations can enable pay-as-you-go overflow or set hard caps to control spend.
  • Reset cycle — quotas reset on your billing anniversary date each month; unused requests do not roll over.

02 Topic 2

Available Models

GitHub Copilot supports a multi-model ecosystem — choose the right model for your task.

  • GPT-4.1 — coding-optimised model with strong instruction following and long-context support up to 1M tokens.
  • Claude Sonnet 4 — excels at nuanced reasoning, refactoring, and multi-file edits with deep code understanding.
  • Gemini 2.5 Pro — Google's advanced model with 1M token context and strong multi-modal capabilities.
  • o3 & o4-mini — OpenAI's reasoning models ideal for complex algorithmic & architectural challenges.
  • Model picker — switch models per-conversation in VS Code via the model selector in chat.

03 Topic 3

Usage & Multipliers

Different models consume your premium quota at different rates — plan strategically for maximum value.

  • Multiplier system — each model has a cost multiplier (1× to 8×) that determines how many premium requests a single interaction consumes.
  • Smart defaults — Copilot auto-selects the most efficient model for routine tasks, preserving your premium quota.
  • Usage dashboard — monitor real-time consumption in GitHub Settings → Copilot → Usage to track burn rate.
  • Org-level controls — admins can restrict model access, set spending limits, and allocate pools across teams.
  • Optimisation tip — use base models for boilerplate and reserve premium models for complex reasoning tasks.
# Usage Strategy Example Monthly Budget: 300 premium requests # Efficient allocation: Daily coding (GPT-4o, 1×) 20 req/day × 10 days = 200 req Code reviews (Claude Sonnet, 1×) 5 req/day × 10 days = 50 req Architecture (o3, 4×) 3 sessions × 4× = 12 req Long analysis (Gemini, 2×) 5 sessions × 2× = 10 req ────────────────────────── Total: 272 / 300 ✓ within budget
