01 Topic 1

Custom Instructions

Shape how Copilot thinks, responds, and generates code — aligned with your standards.

  • Project-level conventions — enforce naming, patterns, and architecture automatically.
  • Persona control — set tone, verbosity, and language preferences.
  • Framework guidance — steer Copilot toward preferred libraries & APIs.
  • Security policies — embed OWASP rules and compliance checks into suggestions.
  • Scoped instructions — apply different rules per file type or folder.
# .github/copilot-instructions.md Always use TypeScript strict mode. Prefer functional components with React hooks over class components. Follow the Airbnb style guide. Never use any type — prefer unknown with type guards. Include unit tests for every public function.
02 Topic 2

Agent Orchestration

Coordinate multiple specialised agents into powerful, multi-step workflows.

  • Multi-agent pipelines — chain agents for plan → code → test → review loops.
  • Tool integration — connect agents to MCP servers, APIs, and external services.
  • Context sharing — pass rich context between agents via structured handoffs.
  • Parallel execution — run independent sub-tasks concurrently for speed.
  • Human-in-the-loop — insert approval gates at critical decision points.
# Orchestration flow User Request → Planner Agent breaks task into steps → Coder Agent implements each step → Reviewer Agent validates & suggests fixes → Deployer Agent ships to staging → ✓ Done
03 Topic 3

Custom Agents

Build domain-specific agents with tailored tools, skills, and knowledge.

  • Agent definition files — declarative YAML/Markdown to define behaviour & tools.
  • Skill libraries — attach reusable domain knowledge as packaged skills.
  • Tool restrictions — limit which tools an agent can access for safety.
  • Custom prompts — fine-tune system prompts for specialised personas.
  • Composability — nest agents within other agents for layered workflows.
# .github/agents/my-agent.agent.md --- name: Database Expert description: SQL & schema design tools: - run_in_terminal - semantic_search - mcp_postgres --- You are a senior database engineer. Always prefer normalized schemas. Suggest indexes for slow queries. Use parameterized queries only.
