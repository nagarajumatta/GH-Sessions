01 Command 1

/plan — Plan Before You Code

Create a structured implementation plan before writing a single line of code. Toggle with Shift+Tab or use the slash command.

  • Structured plans — Copilot analyzes your request, asks clarifying questions, then builds a step-by-step plan.
  • Saved to plan.md — the plan is persisted in your session folder for reference and editing.
  • Approval workflow — review and approve the plan before Copilot starts implementing.
  • Best for complex tasks — multi-file changes, refactoring, new feature implementations.
  • Edit interactively — press Ctrl+Y to open the plan in your default editor.
# Start plan mode with a slash command > /plan Add OAuth2 authentication with Google and GitHub providers # Copilot will: # 1. Analyze your codebase # 2. Ask clarifying questions # 3. Create a structured plan 📋 Plan saved to plan.md - Install dependencies - Create auth routes - Implement passport strategies - Add session management - Write integration tests # Approve and implement > Proceed with the plan
💡 Tip: Press Shift+Tab to toggle plan mode on/off for all prompts.
02 Command 2

/model — Select Your AI Model

Switch between models mid-session based on task complexity — from fast daily coding to deep architectural reasoning.

  • Auto mode — intelligently selects models based on system health; reduces rate limiting and latency.
  • Claude Opus 4.5 — complex architecture, difficult debugging, and nuanced refactoring tasks.
  • Claude Sonnet 4.5 — fast and cost-effective for day-to-day coding and routine tasks.
  • GPT-5.2 Codex — excellent for code generation, code review, and straightforward implementations.
  • Switch anytime — change models mid-session as task complexity changes.
# View and select available models > /model Available models: ● Auto ← recommended ● Opus 4.5 ← deep reasoning ● Sonnet 4.5 ← fast & efficient ● Codex 5.2 ← code generation # Switch to Opus for complex work > /model opus ✓ Switched to Claude Opus 4.5 # Switch to Sonnet for routine tasks > /model sonnet ✓ Switched to Claude Sonnet 4.5
💡 Tip: Use Opus for architecture reviews, Sonnet for everyday work, and Codex as a second opinion on generated code.
03 Command 3

/review — AI Code Review

Get thorough code review assistance — catch bugs, security issues, and style violations before opening a PR.

  • Multi-model review — use multiple models for comprehensive coverage of potential issues.
  • Branch comparison — review changes against main or any target branch.
  • Focus areas — direct the review to focus on bugs, security, performance, or style.
  • Actionable feedback — get specific suggestions, not just high-level comments.
  • Built-in sub-agent — /review runs as a dedicated sub-agent with its own context.
# Review current branch against main > /review Review changes in my current branch against main. Focus on potential bugs and security issues. # Multi-model review for thoroughness > /review Use Opus 4.5 and Codex 5.2 to review changes against `main`. 🔍 Reviewing 12 changed files... ⚠ SQL injection risk in user.ts:42 ⚠ Missing null check in api.ts:87 ✓ auth.ts looks good 💡 Consider adding error boundary
💡 Tip: Run /review before every PR to catch issues early. Combine with /plan for a complete code → review → commit workflow.
04 Command 4

/delegate — Offload to Cloud

Hand off tasks to Copilot's cloud agent — it creates a PR while you keep working locally.

  • Async execution — the cloud agent works in the background while you continue coding.
  • PR creation — the agent creates a pull request with the completed changes.
  • Cross-repo capable — delegate changes to other repositories you have access to.
  • Best for tangential work — documentation updates, refactoring separate modules, boilerplate.
  • Requires GitHub sign-in — delegates to GitHub's server-side Copilot cloud agent.
# Delegate a task to cloud agent > /delegate Add dark mode support to the settings page ☁️ Sending to Copilot cloud agent... ✓ Task accepted — PR incoming # Keep working locally while it runs > Now let's fix the login bug... # Great for tangential tasks: > /delegate Update README with new API endpoints > /delegate Add JSDoc comments to all exported functions
💡 Tip: Use /delegate for tangential tasks and documentation. Keep core feature work and debugging local.
05 Command 5

/fleet — Parallel Task Execution

Break large tasks into parallel subtasks executed by multiple sub-agents simultaneously — dramatically speed up big jobs.

  • Parallel sub-agents — Copilot splits work into independent subtasks and runs them concurrently.
  • Speed multiplier — large migrations, refactoring, and test generation complete much faster.
  • Automatic coordination — sub-agents coordinate to avoid conflicts across files.
  • Autonomous mode — combine with autopilot for fully autonomous parallel execution.
  • Progress tracking — monitor each sub-agent's progress in real time.
# Parallelize a large migration > /fleet Migrate all class components to functional components with hooks 🚀 Breaking into 4 parallel tasks: Agent 1: UserProfile, UserSettings Agent 2: Dashboard, Analytics Agent 3: Auth, Login, Signup Agent 4: Tests for all migrated # Parallelize test generation > /fleet Write unit tests for all services in src/services/ ✓ All 4 agents completed successfully
💡 Tip: Use /fleet at the start of your prompt for any large-scale changes spanning many files.
06 Command 6

/context — Visualize Your Context

See exactly how your context window is being used — system tokens, message history, and remaining free space.

  • Token breakdown — see system/tools tokens, message history, and free space allocation.
  • Buffer visibility — understand how much headroom remains for complex prompts.
  • Infinite sessions — Copilot auto-compacts when context fills up, so you never lose work.
  • Optimization signal — know when to run /compact or /clear for best results.
  • Diagnostic tool — helps explain why responses may become less focused over long sessions.
# Check your context window usage > /context 📊 Context Window Usage ━━━━━━━━━━━━━━━━━━━━━━━━━━━ System & tools: 18,240 tokens Message history: 42,680 tokens Free space: 67,080 tokens Buffer: 4,000 tokens ━━━━━━━━━━━━━━━━━━━━━━━━━━━ Total capacity: 128,000 tokens Usage: 47.6% # Plenty of room to keep going! 🎯
💡 Tip: If context exceeds ~80%, consider running /compact or /clear to improve response quality.
07 Command 7

/session — Session Management

Inspect session state, checkpoints, files, and plans — your window into Copilot's working memory.

  • /session — view current session information and metadata.
  • /session checkpoints — list all compaction checkpoints with summaries.
  • /session checkpoints N — view details of a specific checkpoint.
  • /session files — list temporary artifacts created during the session.
  • /session plan — view the current implementation plan if one exists.
# View current session info > /session Session: a3f2-b8c1 Model: Claude Opus 4.5 Duration: 42 minutes # List checkpoints > /session checkpoints 1. Initial exploration (12 min) 2. Auth implementation (28 min) # View checkpoint details > /session checkpoints 2 # View session files > /session files # View current plan > /session plan
💡 Tip: Session data is stored in ~/.copilot/session-state/{id}/ — useful for auditing and sharing.
08 Command 8

/add-dir — Multi-Repo Workflows

Expand Copilot's access beyond the current directory — work across microservices, monorepos, and related projects.

  • /add-dir <path> — grant Copilot access to an additional repository or folder.
  • /list-dirs — view all currently accessible directories.
  • Cross-repo changes — make coordinated changes across API, frontend, and shared libs.
  • Parent directory trick — start copilot from a parent folder to access all child repos.
  • Microservices friendly — ideal for API contract changes with client updates.
# Add additional repos to the session > /add-dir /home/me/projects/backend > /add-dir /home/me/projects/frontend > /add-dir /home/me/projects/shared-lib # View all accessible directories > /list-dirs 1. /home/me/projects/api-gateway 2. /home/me/projects/backend 3. /home/me/projects/frontend 4. /home/me/projects/shared-lib # Or start from parent directory $ cd ~/projects $ copilot # Access all child repos automatically
💡 Tip: Use @/path/to/repo in your prompts to reference specific repos when working across multiple.
09 Command 9

--allow-tool & --deny-tool

Pre-configure fine-grained tool permissions — control exactly what Copilot can and cannot execute.

  • CLI flags — pass --allow-tool and --deny-tool at launch to preconfigure permissions.
  • Glob patterns — use wildcards like shell(git:*) or shell(npm run:*).
  • /reset-allowed-tools — reset all previously approved tools mid-session.
  • Safety first — Copilot always asks before potentially destructive operations.
  • Team policies — standardize permissions via scripts or shell profiles.
# Allow all git commands, deny push $ copilot \ --allow-tool='shell(git:*)' \ --deny-tool='shell(git push)' # Common permission patterns shell(git:*) # All git cmds shell(npm run:*) # All npm scripts shell(npm run test:*)# npm test only write # Allow file writes # Reset permissions mid-session > /reset-allowed-tools ✓ All tool permissions reset
💡 Tip: Add allow/deny flags to your shell alias for consistent security across all sessions.
10 Command 10

/compact & /clear — Session Hygiene

Keep your sessions focused and performant — compact history, clear context, or start fresh between unrelated tasks.

  • /compact — manually trigger context compaction; summarizes history while preserving key info.
  • /clear — reset context for the current session; ideal between unrelated tasks.
  • /new — start a completely new session, preserving custom instructions.
  • Auto-compaction — Copilot handles this automatically, but manual runs can help long sessions.
  • Better responses — focused context produces higher quality, more relevant output.
# Manually compact context > /compact ✓ Context compacted Summarized 42 messages → 1 summary Freed 38,000 tokens # Clear context between tasks > /clear ✓ Context cleared — ready for new task # Start a brand new session > /new ✓ New session started # Recommended workflow: # Task A → /clear → Task B → /clear # Like fresh conversations 💬
💡 Tip: Use /clear or /new between unrelated tasks — it's like starting a fresh conversation with a colleague.
★ Bonus — Putting It All Together

The Optimal Workflow

Combine these commands into a powerful explore → plan → code → review → commit cycle for maximum productivity.

  • 1. Explore — Read the auth files but don't write code yet
  • 2. Plan — /plan Implement password reset flow
  • 3. Review plan — check the plan, suggest modifications
  • 4. Implement — Proceed with the plan
  • 5. Verify — Run the tests and fix any failures
  • 6. Review — /review Focus on security and bugs
  • 7. Commit — Commit with a descriptive message
  • 8. Clean up — /clear to start next task fresh
# ① Explore the codebase > Read the authentication files but don't write code yet # ② Create a plan > /plan Implement password reset flow # ③ Review & approve plan, then implement > Proceed with the plan # ④ Verify with tests > Run the tests and fix any failures # ⑤ AI code review > /review Focus on security issues # ⑥ Commit and clean up > Commit these changes > /clear
💡 Pro Workflow: Explore → /plan → Implement → /review → Commit → /clear. Rinse and repeat!
